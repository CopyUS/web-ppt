# Web.PPT — Live-Webseiten in PowerPoint

> Zeigen Sie jede beliebige Website waehrend Ihrer PowerPoint-Praesentation — vollstaendig steuerbar mit einem Clicker, ohne die Bildschirmpraesentation jemals zu verlassen.

---

## Warum Web.PPT?

Eine Live-Webseite waehrend einer Praesentation anzuzeigen sollte einfach sein. Leider ist es das nicht:

| Tool | Problem |
|------|---------|
| **LiveWeb VBA** | Verwendete das Internet Explorer ActiveX-Steuerelement — von Microsoft eingestellt. Fing ausserdem den Tastatur-/Mausfokus ab, wodurch es unmoeglich war, die Praesentation mit einem Clicker oder einer Fernbedienung zu steuern. |
| **Microsoft Web Viewer** | Funktioniert nicht im Bildschirmpraesentation-Modus ([bekannte Fehler](https://github.com/OfficeDev/office-js/issues/4410), seit Jahren nicht behoben) |
| **ClassPoint embedded browser** | Im Oktober 2025 entfernt |

**Web.PPT** ist ein kostenloses, quelloffenes Office Add-in, das diese Luecke schliesst. Es funktioniert ueberall, wo PowerPoint laeuft — Windows, Mac, iPad und Web.

### So funktioniert es

1. Sie geben eine URL im Einstellungsbereich (Task Pane) ein
2. Waehrend der Bildschirmpraesentation klicken Sie auf **Show WebPage.PPT** im Menueband
3. Ein Browserfenster oeffnet sich ueber Ihrer Praesentation mit der Live-Website
4. Der automatische Schliesstimer schliesst das Viewer-Fenster automatisch nach der eingestellten Zeit und gibt den Fokus an PowerPoint zurueck — sodass Sie mit einem Clicker oder einer Fernbedienung weiter durch die Folien navigieren koennen

Keine VBA-Makros. Kein ActiveX. Keine Installationsprobleme.

---

## Systemanforderungen

| Anforderung | Details |
|-------------|---------|
| **PowerPoint-Version** | **Microsoft 365** (Abonnement) oder **Office 2021** (Einmalkauf) |
| **Betriebssystem** | Windows 10/11, macOS, iPadOS oder ein beliebiges Betriebssystem mit Browser (PowerPoint Online) |
| **Internetverbindung** | Erforderlich zum Laden von Webseiten |

### Warum Office 2019 und aelter nicht unterstuetzt werden

Web.PPT verwendet die PowerPoint JavaScript API (`PowerPoint.run`, `getSelectedSlides`), um die aktuelle Folie zu erkennen und folienspezifische Einstellungen zu speichern. Diese API (Anforderungssatz **PowerPointApi 1.1+**) ist nur verfuegbar in:

- Microsoft 365 (alle Updatekanaele)
- Office 2021 (Dauerlizenz)

Office 2019, 2016 und aelter enthalten diese APIs nicht, sodass das Add-in **nicht feststellen kann, welche Folie aktiv ist**, und nicht korrekt funktioniert.

## Installation

### Option 1: AppSource (Microsoft 365)

> Diese Option wird moeglicherweise verfuegbar, nachdem das Add-in im AppSource veroeffentlicht wurde.

Funktioniert auf allen Plattformen (Windows, Mac, iPad, Web):

1. Oeffnen Sie PowerPoint → Registerkarte **Einfuegen** → **Add-Ins abrufen**
2. Suchen Sie nach **"Web.PPT"**
3. Klicken Sie auf **Hinzufuegen** — das Add-in erscheint sofort im Menueband

---

### Option 2: Sideloading — PowerPoint 2021 / 365 Desktop (Windows)

Verwenden Sie diese Methode, um das Add-in aus einer lokalen Datei oder einem freigegebenen Ordner zu installieren. Dies ist das grundlegende Szenario, wenn AppSource nicht verfuegbar ist.

#### Schritt 1: Registerkarte "Entwicklertools" im Menueband aktivieren

Die Registerkarte "Entwicklertools" wird fuer Sideloading von Add-ins benoetigt. Falls Sie sie im Menueband nicht sehen:

1. Oeffnen Sie PowerPoint → **Datei** → **Optionen** → **Menueband anpassen**
2. Aktivieren Sie in der rechten Spalte ("Hauptregisterkarten") das Kontrollkaestchen **Entwicklertools**
3. Klicken Sie auf **OK**

Die Registerkarte **Entwicklertools** erscheint nun im Menueband.

#### Schritt 2: Manifest herunterladen

Laden Sie `manifest-gh.xml` von der [Release-Seite](https://github.com/CopyUS) oder aus dem Ordner `dist/` dieses Repositorys herunter.

#### Schritt 3: Freigegebenen Ordner erstellen

1. Erstellen Sie einen Ordner auf Ihrem Computer, zum Beispiel:
   ```
   C:\OfficeAddins\
   ```
2. Legen Sie `manifest-gh.xml` in diesen Ordner
3. Rechtsklicken Sie auf den Ordner → **Eigenschaften** → Registerkarte **Freigabe** → **Freigeben...**
4. Geben Sie den Ordner fuer sich selbst frei (oder behalten Sie die Standardfreigabeeinstellungen bei)
5. Kopieren Sie den **Netzwerkpfad** (z. B. `\\IHRPC\OfficeAddins`)

#### Schritt 4: Ordner als vertrauenswuerdigen Katalog hinzufuegen

1. Oeffnen Sie PowerPoint → **Datei** → **Optionen** → **Vertrauensstellungscenter** → **Einstellungen fuer das Vertrauensstellungscenter...**
2. Waehlen Sie im linken Menue **Vertrauenswuerdige Add-In-Kataloge**
3. Fuegen Sie im Feld **Katalog-URL** den Netzwerkpfad aus Schritt 3 ein (z. B. `\\IHRPC\OfficeAddins`)
4. Klicken Sie auf **Katalog hinzufuegen**
5. Aktivieren Sie das Kontrollkaestchen **Im Menue anzeigen** fuer den hinzugefuegten Katalog
6. Klicken Sie auf **OK** → **OK**, um alle Dialoge zu schliessen
7. **Starten Sie PowerPoint neu** (schliessen Sie alle PowerPoint-Fenster vollstaendig)

#### Schritt 5: Add-in laden

1. Starten Sie PowerPoint neu (schliessen Sie es vollstaendig und oeffnen Sie es erneut)
2. Gehen Sie zur Registerkarte **Entwicklertools** → klicken Sie auf **Add-Ins**
3. Waehlen Sie oben die Registerkarte **Freigegebener Ordner** (Shared Folder)
4. Sie sehen **Web.PPT** — klicken Sie auf **Hinzufuegen** (Add)
5. Warten Sie einige Sekunden — die Schaltflaechen **Add WebPage.PPT** und **Show WebPage.PPT** erscheinen auf der Registerkarte **Einfuegen**

> **Wichtig:** Die Schaltflaechen erscheinen **nicht automatisch** im Menueband nach dem Hinzufuegen des vertrauenswuerdigen Ordners. Sie muessen das Add-In manuell ueber **Entwicklertools → Add-Ins → Freigegebener Ordner** laden. Dies ist eine Besonderheit der Sideloading-Installation — Sie muessen das Add-In bei jedem Neustart von PowerPoint erneut hinzufuegen. Nach der Veroeffentlichung im AppSource wird das Add-In automatisch geladen.

#### Wenn etwas schief gelaufen ist

| Problem | Loesung |
|---------|---------|
| Add-In wird im „Freigegebenen Ordner" nicht angezeigt | Ueberpruefen Sie den Netzwerkpfad in den Einstellungen fuer vertrauenswuerdige Kataloge. Stellen Sie sicher, dass **„Im Menue anzeigen"** aktiviert ist. Schliessen Sie PowerPoint vollstaendig und oeffnen Sie es erneut. |
| Registerkarte „Freigegebener Ordner" ist leer | Schliessen Sie **alle** PowerPoint-Fenster, warten Sie 10 Sekunden und oeffnen Sie es erneut. Manchmal sind 2-3 Neustarts erforderlich. |
| „Add-in Error" beim Klicken auf eine Schaltflaeche | Leeren Sie den Office-Cache (siehe Abschnitt Cache-Bereinigung unten), starten Sie PowerPoint neu und fuegen Sie das Add-In erneut hinzu. |
| Schaltflaechen erschienen, reagieren aber nicht | Warten Sie 5-10 Sekunden nach dem Laden — Office laedt Dateien vom Server herunter. Ueberpruefen Sie Ihre Internetverbindung. |

### Option 3: Sideloading — PowerPoint Online

1. Gehen Sie zu [office.com](https://www.office.com/) → oeffnen Sie PowerPoint → oeffnen Sie eine beliebige Praesentation
2. Registerkarte **Einfuegen** → **Add-Ins** → **Mein Add-In hochladen**
3. Klicken Sie auf **Durchsuchen...** und waehlen Sie die Datei `manifest.json` (JSON-Format, nicht XML)
4. Klicken Sie auf **Hochladen** — das Add-in wird sofort geladen

> **Hinweis:** Per Sideloading geladene Add-ins in PowerPoint Online bleiben nur fuer die aktuelle Sitzung erhalten. Nach dem Aktualisieren der Seite muessen Sie das Add-in erneut hochladen.

---

### Option 4: Sideloading — Mac (Desktop)

1. Laden Sie `manifest-gh.xml` herunter
2. Oeffnen Sie den Finder und navigieren Sie zu:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
   > Falls der Ordner `wef` nicht existiert, erstellen Sie ihn manuell.
3. Kopieren Sie `manifest-gh.xml` in den Ordner `wef`
4. Oeffnen (oder starten) Sie PowerPoint neu
5. Gehen Sie zu **Einfuegen** → **Meine Add-Ins** → **Meine Organisation** — Sie sollten **Web.PPT** sehen

> **Tipp:** Um den Library-Ordner zu oeffnen, druecken Sie **Cmd + Umschalt + G** im Finder und fuegen Sie den Pfad ein.

## Office-Cache leeren (nach Aktualisierungen)

Wenn Sie die Add-in-Dateien aktualisieren (neue `manifest-gh.xml` oder aktualisierte gehostete Seiten), verwendet Office moeglicherweise weiterhin die zwischengespeicherte alte Version. Leeren Sie den Cache, um ein Neuladen zu erzwingen.

### Windows

1. Schliessen Sie **alle** Office-Anwendungen (PowerPoint, Word, Excel usw.)
2. Oeffnen Sie den Datei-Explorer und navigieren Sie zu:
   ```
   %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\
   ```
   > **Tipp:** Druecken Sie **Win + R**, fuegen Sie den Pfad ein und druecken Sie die Eingabetaste.
3. Loeschen Sie **alle Dateien und Ordner** innerhalb des Ordners `Wef`
4. Starten Sie PowerPoint neu und fuegen Sie das Add-in bei Bedarf erneut hinzu

### Mac

1. Schliessen Sie alle Office-Anwendungen
2. Oeffnen Sie den Finder → **Gehe zu** → **Gehe zum Ordner...** (Cmd + Umschalt + G)
3. Navigieren Sie zu:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Library/Caches/
   ```
4. Loeschen Sie den Inhalt des Ordners `Caches`
5. Entfernen und kopieren Sie das Manifest ebenfalls erneut in:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
6. Starten Sie PowerPoint neu

### PowerPoint Online

1. Oeffnen Sie die Browser-Entwicklertools (**F12**)
2. Gehen Sie zur Registerkarte **Application** → **Storage** → **Clear site data**
3. Aktualisieren Sie die Seite und laden Sie das Manifest erneut hoch

## Verwendung

### Eine URL fuer eine Folie einrichten

Jede Folie kann ihre eigene URL haben. Waehlen Sie eine Folie aus, oeffnen Sie das Task Pane (**Einfuegen** → **Add WebPage.PPT**) und geben Sie die URL ein.

Verfuegbare Einstellungen:

| Einstellung | Beschreibung | Standard |
|-------------|-------------|----------|
| **URL** | Die anzuzeigende Webadresse | *(keine)* |
| **Zoom** | Seitenmassstab (50 %–300 %) | 100 % |
| **Fensterbreite** | Breite des Viewer-Fensters (30 %–100 % des Bildschirms) | 80 % |
| **Fensterhoehe** | Hoehe des Viewer-Fensters (30 %–100 % des Bildschirms) | 80 % |
| **Automatisch oeffnen** | Viewer automatisch oeffnen, wenn diese Folie angezeigt wird | Aus |
| **Automatisch schliessen** | Viewer automatisch nach einer festgelegten Zeit schliessen (0–3600 s) | Aus |

Klicken Sie auf **Als Standard festlegen**, um Ihre bevorzugte Groesse/Zoom-Einstellung fuer alle neuen Folien zu speichern.

### Eine Webseite waehrend der Bildschirmpraesentation anzeigen

1. Starten Sie Ihre Bildschirmpraesentation wie gewohnt (**F5** oder Registerkarte **Bildschirmpraesentation**)
2. Klicken Sie auf **Show WebPage.PPT** im Menueband — ein Browserfenster oeffnet sich mit Ihrer Website
3. Schliessen Sie das Fenster (oder warten Sie auf den automatischen Schliesstimer), um zu den Folien zurueckzukehren

### Sprache aendern

Web.PPT unterstuetzt **10 Sprachen**: Englisch, Chinesisch (中文), Spanisch, Deutsch, Franzoesisch, Italienisch, Arabisch, Portugiesisch, Hindi und Russisch. Verwenden Sie das Sprach-Dropdown im Task Pane-Header — die Oberflaeche aktualisiert sich sofort, kein Neustart erforderlich.

## Fuer Ihre eigenen Websites

Web.PPT laedt Websites innerhalb eines iframes. Die meisten persoenlichen Websites und Projektseiten funktionieren sofort. Wenn Ihre Website das Einbetten blockiert, wird eine hilfreiche Meldung mit einer Loesung angezeigt.

Fuegen Sie **eine Zeile** zu Ihrer Serverkonfiguration hinzu:

**Nginx**
```nginx
add_header Content-Security-Policy "frame-ancestors *";
```

**Apache** (`.htaccess` oder Konfiguration)
```apache
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
```

**Node.js / Express**
```javascript
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
```

Starten Sie Ihren Server nach der Aenderung neu. Web.PPT verfuegt auch ueber eine integrierte Einrichtungsanleitung — klicken Sie auf **"Site not loading?"** im Task Pane, wenn eine Website blockiert wird.

**Kein Serverzugriff?** Klicken Sie auf **"Open directly"** im Viewer — die Seite wird in einem vollstaendigen Browserfenster ohne iframe-Einschraenkungen geoeffnet.

## Unterstuetzte Plattformen

| Plattform | Status | Hinweise |
|-----------|--------|----------|
| **Windows 10/11** (Desktop) | Unterstuetzt | Voller Funktionsumfang ueber Sideloading mit freigegebenem Ordner oder AppSource |
| **Mac** (Desktop) | Unterstuetzt | Sideloading ueber den Ordner `~/Library/.../wef/` oder AppSource |
| **iPad** | Unterstuetzt | Nur ueber AppSource; Dialogverhalten kann variieren |
| **PowerPoint Online** | Unterstuetzt | Sideloading per Upload; automatisches Oeffnen bei Folienwechsel nicht verfuegbar |

## Bekannte Einschraenkungen

- **Drittanbieter-Websites, die iframes blockieren** (z. B. Google, Facebook), koennen nicht eingebettet werden. Verwenden Sie den Fallback "Open directly".
- **PowerPoint Online-Bildschirmpraesentation:** Das Ereignis `ActiveViewChanged` wird nicht ausgeloest, daher ist das automatische Oeffnen bei Folienwechsel nicht verfuegbar. Verwenden Sie die Menueband-Schaltflaeche manuell.
- **Office 2019 und aelter:** Nicht unterstuetzt — die erforderliche PowerPoint JavaScript API ist nicht verfuegbar (siehe [Systemanforderungen](#systemanforderungen)).
- Der Viewer oeffnet sich in einem **separaten Browserfenster**, nicht innerhalb des Folienbereichs. Dies ist eine Plattformbeschraenkung von Office Web Add-ins (die Alternative — Content Add-in — ist [im Bildschirmpraesentation-Modus fehlerhaft](https://github.com/OfficeDev/office-js/issues/4410)).

# Web.PPT — Live Web Pages in PowerPoint

> Show any website during your PowerPoint presentation — fully controllable with a clicker, without ever leaving the slideshow.

---

## Why Web.PPT?

Showing a live web page during a presentation should be simple. Unfortunately, it isn't:

| Tool | Problem |
|------|---------|
| **LiveWeb VBA** | Used Internet Explorer ActiveX control — discontinued by Microsoft. Also captures keyboard/mouse focus, making it impossible to control the presentation with a clicker or remote. |
| **Microsoft Web Viewer** | Broken in slideshow mode ([known bugs](https://github.com/OfficeDev/office-js/issues/4410), unfixed for years) |
| **ClassPoint embedded browser** | Removed in October 2025 |

**Web.PPT** is a free, open-source Office Add-in that fills this gap. It works everywhere PowerPoint runs — Windows, Mac, iPad, and Web.

### How it works

1. You enter a URL in the settings panel (Task Pane)
2. During the slideshow, you click **Show WebPage.PPT** on the ribbon
3. A browser window opens on top of your presentation with the live website
4. The auto-close timer closes the viewer window automatically after the set time, returning focus to PowerPoint — so you can continue navigating slides with a clicker or remote control

No VBA macros. No ActiveX. No installation headaches.

## System Requirements

| Requirement | Details |
|-------------|---------|
| **PowerPoint version** | **Microsoft 365** (subscription) or **Office 2021** (one-time purchase) |
| **Operating system** | Windows 10/11, macOS, iPadOS, or any OS with a browser (PowerPoint Online) |
| **Internet connection** | Required to load web pages |

### Why Office 2019 and older are not supported

Web.PPT uses the PowerPoint JavaScript API (`PowerPoint.run`, `getSelectedSlides`) to detect the current slide and save per-slide settings. This API (requirement set **PowerPointApi 1.1+**) is only available in:

- Microsoft 365 (all update channels)
- Office 2021 (perpetual license)

Office 2019, 2016, and earlier do not include these APIs, so the add-in **cannot determine which slide is active** and will not function correctly.

## Installation

### Option 1: AppSource (Microsoft 365)

> This option will be available after the add-in is published to AppSource.

The easiest way — works on all platforms (Windows, Mac, iPad, Web):

1. Open PowerPoint → **Insert** tab → **Get Add-ins**
2. Search for **"Web.PPT"**
3. Click **Add** — the add-in appears on the ribbon immediately

---

### Option 2: Sideloading — PowerPoint 2021 / 365 Desktop (Windows)

Use this method to install the add-in from a local file or shared folder. This is the recommended way for testing or when AppSource is not available.

#### Step 1: Enable the Developer tab on the ribbon

The Developer tab is required for sideloading add-ins. If you don't see it on the ribbon:

1. Open PowerPoint → **File** → **Options** → **Customize Ribbon**
2. In the right column ("Main Tabs"), check the **Developer** checkbox
3. Click **OK**

The **Developer** tab now appears on the ribbon.

#### Step 2: Download the manifest

Download `manifest-gh.xml` from the [releases page](https://github.com/CopyUS) or from the `dist/` folder of this repository.

#### Step 3: Create a shared folder

1. Create a folder on your computer, for example:
   ```
   C:\OfficeAddins\
   ```
2. Place `manifest-gh.xml` into this folder
3. Right-click the folder → **Properties** → **Sharing** tab → **Share...**
4. Share the folder with yourself (or keep the default sharing settings)
5. Copy the **network path** (e.g., `\\YOURPC\OfficeAddins`)

#### Step 4: Add the folder as a Trusted Catalog

1. Open PowerPoint → **File** → **Options** → **Trust Center** → **Trust Center Settings...**
2. Select **Trusted Add-in Catalogs** in the left menu
3. In the **Catalog URL** field, paste the network path from Step 3 (e.g., `\\YOURPC\OfficeAddins`)
4. Click **Add catalog**
5. Check the **Show in Menu** checkbox for the added catalog
6. Click **OK** → **OK** to close all dialogs
7. **Restart PowerPoint** (close all PowerPoint windows completely)

#### Step 5: Load the add-in

1. Open PowerPoint → **Insert** tab → **Get Add-ins** (or **My Add-ins**)
2. Select the **Shared Folder** tab at the top
3. You should see **Web.PPT** — click **Add**
4. The add-in buttons appear on the **Insert** tab

> **Tip:** If the add-in doesn't appear in Shared Folder, double-check the network path and restart PowerPoint.

---

### Option 3: Sideloading — PowerPoint Online

1. Go to [office.com](https://www.office.com/) → open PowerPoint → open any presentation
2. **Insert** tab → **Add-ins** → **Upload My Add-in**
3. Click **Browse...** and select the `manifest.json` file (JSON format, not XML)
4. Click **Upload** — the add-in loads immediately

> **Note:** Sideloaded add-ins in PowerPoint Online persist only for the current session. You will need to re-upload after refreshing the page.

---

### Option 4: Sideloading — Mac (Desktop)

1. Download `manifest-gh.xml`
2. Open Finder and navigate to:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
   > If the `wef` folder doesn't exist, create it manually.
3. Copy `manifest-gh.xml` into the `wef` folder
4. Open (or restart) PowerPoint
5. Go to **Insert** → **My Add-ins** → **My Organization** — you should see **Web.PPT**

> **Tip:** To open the Library folder, press **Cmd + Shift + G** in Finder and paste the path.

## Clearing the Office Cache (after updates)

When you update the add-in files (new `manifest-gh.xml` or updated hosted pages), Office may still use the cached old version. Clear the cache to force a reload.

### Windows

1. Close **all** Office applications (PowerPoint, Word, Excel, etc.)
2. Open File Explorer and navigate to:
   ```
   %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\
   ```
   > **Tip:** Press **Win + R**, paste the path, and press Enter.
3. Delete **all files and folders** inside the `Wef` folder
4. Restart PowerPoint and re-add the add-in if needed

### Mac

1. Close all Office applications
2. Open Finder → **Go** → **Go to Folder...** (Cmd + Shift + G)
3. Navigate to:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Library/Caches/
   ```
4. Delete the contents of the `Caches` folder
5. Also remove and re-copy the manifest in:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
6. Restart PowerPoint

### PowerPoint Online

1. Open browser DevTools (**F12**)
2. Go to **Application** tab → **Storage** → **Clear site data**
3. Refresh the page and re-upload the manifest

## Usage

### Setting up a URL for a slide

Each slide can have its own URL. Select a slide, open the Task Pane (**Insert** → **Add WebPage.PPT**), and enter the URL.

Available settings:

| Setting | Description | Default |
|---------|-------------|---------|
| **URL** | The web address to display | *(none)* |
| **Zoom** | Scale the page (50 %–300 %) | 100 % |
| **Window width** | Viewer window width (30 %–100 % of screen) | 80 % |
| **Window height** | Viewer window height (30 %–100 % of screen) | 80 % |
| **Auto-open** | Automatically open the viewer when this slide is shown | Off |
| **Auto-close** | Automatically close the viewer after a set time (0–3600 s) | Off |

Click **Set as defaults** to save your preferred size/zoom for all new slides.

### Showing a web page during a slideshow

1. Start your slideshow normally (**F5** or **Slide Show** tab)
2. Click **Show WebPage.PPT** on the ribbon — a browser window opens with your site
3. Close the window (or wait for auto-close timer) to return to slides

### Changing the language

Web.PPT supports **10 languages**: English, Chinese (中文), Spanish, German, French, Italian, Arabic, Portuguese, Hindi, and Russian. Use the language dropdown in the Task Pane header — the interface updates instantly, no restart needed.

## For Your Own Sites

Web.PPT loads websites inside an iframe. Most personal and project sites work out of the box. If your site blocks embedding, you'll see a friendly message with a fix.

Add **one line** to your server config:

**Nginx**
```nginx
add_header Content-Security-Policy "frame-ancestors *";
```

**Apache** (`.htaccess` or config)
```apache
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
```

**Node.js / Express**
```javascript
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
```

Restart your server after the change. Web.PPT also has a built-in setup guide — click **"Site not loading?"** in the Task Pane when a site is blocked.

**Don't have server access?** Click **"Open directly"** in the viewer — it opens the page in a full browser window without iframe restrictions.

## Known Limitations

- **Third-party sites that block iframes** (e.g., Google, Facebook) cannot be embedded. Use the "Open directly" fallback.
- **PowerPoint Online slideshow:** The `ActiveViewChanged` event doesn't fire, so auto-open on slide change is not available. Use the ribbon button manually.
- **Office 2019 and older:** Not supported — the required PowerPoint JavaScript API is not available (see [System Requirements](#system-requirements)).
- The viewer opens in a **separate browser window**, not inside the slide area. This is a platform limitation of Office Web Add-ins (the alternative — Content Add-in — is [broken in slideshow mode](https://github.com/OfficeDev/office-js/issues/4410)).

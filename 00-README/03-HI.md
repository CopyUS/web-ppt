# Web.PPT — PowerPoint में लाइव वेब पेज

> अपनी PowerPoint प्रेजेंटेशन के दौरान कोई भी वेबसाइट दिखाएं — क्लिकर से पूरी तरह नियंत्रित, स्लाइडशो छोड़े बिना।

---

## Web.PPT क्यों?

प्रेजेंटेशन के दौरान लाइव वेब पेज दिखाना आसान होना चाहिए। दुर्भाग्यवश, ऐसा नहीं है:

| टूल | समस्या |
|------|---------|
| **LiveWeb VBA** | Internet Explorer ActiveX कंट्रोल का उपयोग करता था — Microsoft ने इसे बंद कर दिया। साथ ही यह कीबोर्ड/माउस फोकस कैप्चर कर लेता था, जिससे क्लिकर या रिमोट कंट्रोल से प्रेजेंटेशन चलाना असंभव हो जाता था। |
| **Microsoft Web Viewer** | स्लाइडशो मोड में काम नहीं करता ([ज्ञात बग](https://github.com/OfficeDev/office-js/issues/4410), वर्षों से ठीक नहीं किया गया) |
| **ClassPoint embedded browser** | अक्टूबर 2025 में हटा दिया गया |

**Web.PPT** एक मुफ्त, ओपन-सोर्स Office Add-in है जो इस कमी को पूरा करता है। यह हर जगह काम करता है जहाँ PowerPoint चलता है — Windows, Mac, iPad, और Web।

### यह कैसे काम करता है

1. आप सेटिंग्स पैनल (Task Pane) में एक URL दर्ज करते हैं
2. स्लाइडशो के दौरान, आप रिबन पर **Show WebPage.PPT** क्लिक करते हैं
3. आपकी प्रेजेंटेशन के ऊपर एक ब्राउज़र विंडो लाइव वेबसाइट के साथ खुलती है
4. ऑटो-क्लोज़ टाइमर निर्धारित समय के बाद व्यूअर विंडो को स्वचालित रूप से बंद कर देता है, फोकस PowerPoint पर वापस लौटाता है — ताकि आप क्लिकर या रिमोट कंट्रोल से स्लाइड्स नेविगेट करना जारी रख सकें

कोई VBA मैक्रोज़ नहीं। कोई ActiveX नहीं। इंस्टॉलेशन की कोई परेशानी नहीं।

## सिस्टम आवश्यकताएं

| आवश्यकता | विवरण |
|-------------|---------|
| **PowerPoint संस्करण** | **Microsoft 365** (सब्सक्रिप्शन) या **Office 2021** (एकमुश्त खरीदारी) |
| **ऑपरेटिंग सिस्टम** | Windows 10/11, macOS, iPadOS, या कोई भी OS जिसमें ब्राउज़र हो (PowerPoint Online) |
| **इंटरनेट कनेक्शन** | वेब पेज लोड करने के लिए आवश्यक |

### Office 2019 और पुराने संस्करण क्यों समर्थित नहीं हैं

Web.PPT वर्तमान स्लाइड का पता लगाने और प्रति-स्लाइड सेटिंग्स सहेजने के लिए PowerPoint JavaScript API (`PowerPoint.run`, `getSelectedSlides`) का उपयोग करता है। यह API (requirement set **PowerPointApi 1.1+**) केवल इनमें उपलब्ध है:

- Microsoft 365 (सभी अपडेट चैनल)
- Office 2021 (परमानेंट लाइसेंस)

Office 2019, 2016, और उससे पुराने संस्करणों में ये API शामिल नहीं हैं, इसलिए ऐड-इन **यह निर्धारित नहीं कर सकता कि कौन सी स्लाइड सक्रिय है** और सही ढंग से काम नहीं करेगा।

## इंस्टॉलेशन

### विकल्प 1: AppSource (Microsoft 365)

> यह विकल्प ऐड-इन के AppSource पर प्रकाशित होने के बाद उपलब्ध होगा।

सबसे आसान तरीका — सभी प्लेटफ़ॉर्म पर काम करता है (Windows, Mac, iPad, Web):

1. PowerPoint खोलें → **सम्मिलित करें** टैब → **ऐड-इन प्राप्त करें**
2. **"Web.PPT"** खोजें
3. **Add** क्लिक करें — ऐड-इन तुरंत रिबन पर दिखाई देगा

---

### विकल्प 2: Sideloading — PowerPoint 2021 / 365 Desktop (Windows)

इस विधि का उपयोग लोकल फ़ाइल या शेयर्ड फ़ोल्डर से ऐड-इन इंस्टॉल करने के लिए करें। यह टेस्टिंग के लिए या जब AppSource उपलब्ध नहीं हो तब अनुशंसित तरीका है।

#### चरण 1: रिबन पर डेवलपर टैब सक्षम करें

Sideloading ऐड-इन्स के लिए डेवलपर टैब आवश्यक है। यदि आपको रिबन पर यह दिखाई नहीं देता:

1. PowerPoint खोलें → **फ़ाइल** → **विकल्प** → **रिबन अनुकूलित करें**
2. दाएं कॉलम ("मुख्य टैब") में, **डेवलपर** चेकबॉक्स पर टिक करें
3. **OK** क्लिक करें

अब **डेवलपर** टैब रिबन पर दिखाई देगा।

#### चरण 2: मैनिफ़ेस्ट डाउनलोड करें

[releases पेज](https://github.com/CopyUS) से या इस रिपॉज़िटरी के `dist/` फ़ोल्डर से `manifest-gh.xml` डाउनलोड करें।

#### चरण 3: शेयर्ड फ़ोल्डर बनाएं

1. अपने कंप्यूटर पर एक फ़ोल्डर बनाएं, उदाहरण के लिए:
   ```
   C:\OfficeAddins\
   ```
2. `manifest-gh.xml` को इस फ़ोल्डर में रखें
3. फ़ोल्डर पर राइट-क्लिक करें → **Properties** → **Sharing** टैब → **Share...**
4. फ़ोल्डर को अपने साथ शेयर करें (या डिफ़ॉल्ट शेयरिंग सेटिंग्स रखें)
5. **नेटवर्क पाथ** कॉपी करें (उदा., `\\YOURPC\OfficeAddins`)

#### चरण 4: फ़ोल्डर को विश्वसनीय कैटलॉग के रूप में जोड़ें

1. PowerPoint खोलें → **फ़ाइल** → **विकल्प** → **विश्वास केंद्र** → **विश्वास केंद्र सेटिंग्स...**
2. बाएं मेनू में **Trusted Add-in Catalogs** चुनें
3. **Catalog URL** फ़ील्ड में, चरण 3 से नेटवर्क पाथ पेस्ट करें (उदा., `\\YOURPC\OfficeAddins`)
4. **Add catalog** क्लिक करें
5. जोड़े गए कैटलॉग के लिए **Show in Menu** चेकबॉक्स पर टिक करें
6. सभी डायलॉग बंद करने के लिए **OK** → **OK** क्लिक करें
7. **PowerPoint रीस्टार्ट करें** (सभी PowerPoint विंडो पूरी तरह बंद करें)

#### चरण 5: ऐड-इन लोड करें

1. PowerPoint को पुनः आरंभ करें (पूरी तरह बंद करके दोबारा खोलें)
2. **डेवलपर** (Developer) टैब पर जाएं → **ऐड-इन्स** (Add-ins) पर क्लिक करें
3. शीर्ष पर **शेयर्ड फ़ोल्डर** (Shared Folder) टैब चुनें
4. आपको **Web.PPT** दिखेगा — **जोड़ें** (Add) पर क्लिक करें
5. कुछ सेकंड प्रतीक्षा करें — **Add WebPage.PPT** और **Show WebPage.PPT** बटन **सम्मिलित करें** (Insert) टैब पर दिखाई देंगे

> **महत्वपूर्ण:** विश्वसनीय फ़ोल्डर जोड़ने के बाद बटन रिबन पर **स्वचालित रूप से नहीं दिखते**। आपको **डेवलपर → ऐड-इन्स → शेयर्ड फ़ोल्डर** के माध्यम से ऐड-इन को मैन्युअल रूप से लोड करना होगा। यह साइडलोडिंग इंस्टॉलेशन की विशेषता है — हर बार PowerPoint पुनः आरंभ करने पर ऐड-इन को दोबारा जोड़ना होगा। AppSource पर प्रकाशन के बाद ऐड-इन स्वचालित रूप से लोड होगा।

#### समस्या निवारण

| समस्या | समाधान |
|---------|--------|
| ऐड-इन Shared Folder में दिखाई नहीं देता | नेटवर्क पाथ दोबारा जांचें (चरण 3), सुनिश्चित करें कि **Show in Menu** चेकबॉक्स चालू है (चरण 4), और PowerPoint को पूरी तरह बंद करके दोबारा खोलें |
| शेयर्ड फ़ोल्डर टैब खाली है | `manifest-gh.xml` फ़ाइल सही फ़ोल्डर में है यह जांचें; फ़ोल्डर शेयरिंग सेटिंग्स सत्यापित करें; File Explorer में `\\YOURPC\OfficeAddins` पाथ खोलकर पहुंच की पुष्टि करें |
| ऐड-इन जोड़ते समय त्रुटि आती है | Office कैश साफ़ करें (नीचे निर्देश देखें), `manifest-gh.xml` फ़ाइल दूषित नहीं है यह जांचें, और PowerPoint रीस्टार्ट करें |
| बटन रिबन पर दिखते हैं लेकिन प्रतिक्रिया नहीं देते | PowerPoint को पूरी तरह बंद करें (सभी विंडो), Office कैश साफ़ करें, और ऐड-इन दोबारा लोड करें; सुनिश्चित करें कि इंटरनेट कनेक्शन सक्रिय है |

### विकल्प 3: Sideloading — PowerPoint Online

1. [office.com](https://www.office.com/) पर जाएं → PowerPoint खोलें → कोई भी प्रेजेंटेशन खोलें
2. **सम्मिलित करें** टैब → **Add-ins** → **Upload My Add-in**
3. **Browse...** क्लिक करें और `manifest.json` फ़ाइल चुनें (JSON फ़ॉर्मेट, XML नहीं)
4. **Upload** क्लिक करें — ऐड-इन तुरंत लोड हो जाएगा

> **नोट:** PowerPoint Online में Sideloaded ऐड-इन्स केवल वर्तमान सत्र तक रहते हैं। पेज रिफ़्रेश करने के बाद आपको दोबारा अपलोड करना होगा।

---

### विकल्प 4: Sideloading — Mac (Desktop)

1. `manifest-gh.xml` डाउनलोड करें
2. Finder खोलें और इस पाथ पर जाएं:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
   > यदि `wef` फ़ोल्डर मौजूद नहीं है, तो इसे मैन्युअली बनाएं।
3. `manifest-gh.xml` को `wef` फ़ोल्डर में कॉपी करें
4. PowerPoint खोलें (या रीस्टार्ट करें)
5. **सम्मिलित करें** → **My Add-ins** → **My Organization** पर जाएं — आपको **Web.PPT** दिखाई देगा

> **सुझाव:** Library फ़ोल्डर खोलने के लिए, Finder में **Cmd + Shift + G** दबाएं और पाथ पेस्ट करें।

## Office कैश साफ़ करना (अपडेट के बाद)

जब आप ऐड-इन फ़ाइलें अपडेट करते हैं (नया `manifest-gh.xml` या अपडेटेड होस्टेड पेज), तो Office पुराने कैश्ड संस्करण का उपयोग कर सकता है। रीलोड करने के लिए कैश साफ़ करें।

### Windows

1. **सभी** Office एप्लिकेशन बंद करें (PowerPoint, Word, Excel, आदि)
2. File Explorer खोलें और इस पाथ पर जाएं:
   ```
   %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\
   ```
   > **सुझाव:** **Win + R** दबाएं, पाथ पेस्ट करें, और Enter दबाएं।
3. `Wef` फ़ोल्डर के अंदर की **सभी फ़ाइलें और फ़ोल्डर** डिलीट करें
4. PowerPoint रीस्टार्ट करें और यदि आवश्यक हो तो ऐड-इन दोबारा जोड़ें

### Mac

1. सभी Office एप्लिकेशन बंद करें
2. Finder खोलें → **Go** → **Go to Folder...** (Cmd + Shift + G)
3. इस पाथ पर जाएं:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Library/Caches/
   ```
4. `Caches` फ़ोल्डर की सामग्री डिलीट करें
5. इसमें मैनिफ़ेस्ट भी हटाएं और दोबारा कॉपी करें:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
6. PowerPoint रीस्टार्ट करें

### PowerPoint Online

1. ब्राउज़र DevTools खोलें (**F12**)
2. **Application** टैब → **Storage** → **Clear site data** पर जाएं
3. पेज रिफ़्रेश करें और मैनिफ़ेस्ट दोबारा अपलोड करें

## उपयोग

### किसी स्लाइड के लिए URL सेट करना

प्रत्येक स्लाइड का अपना URL हो सकता है। एक स्लाइड चुनें, Task Pane खोलें (**सम्मिलित करें** → **Add WebPage.PPT**), और URL दर्ज करें।

उपलब्ध सेटिंग्स:

| सेटिंग | विवरण | डिफ़ॉल्ट |
|---------|-------------|---------|
| **URL** | प्रदर्शित करने का वेब पता | *(कोई नहीं)* |
| **ज़ूम** | पेज का स्केल (50 %–300 %) | 100 % |
| **विंडो चौड़ाई** | व्यूअर विंडो की चौड़ाई (स्क्रीन का 30 %–100 %) | 80 % |
| **विंडो ऊंचाई** | व्यूअर विंडो की ऊंचाई (स्क्रीन का 30 %–100 %) | 80 % |
| **ऑटो-ओपन** | जब यह स्लाइड दिखाई जाए तो व्यूअर स्वचालित रूप से खोलें | बंद |
| **ऑटो-क्लोज़** | निर्धारित समय के बाद व्यूअर स्वचालित रूप से बंद करें (0–3600 s) | बंद |

सभी नई स्लाइड्स के लिए अपना पसंदीदा आकार/ज़ूम सहेजने के लिए **Set as defaults** क्लिक करें।

### स्लाइडशो के दौरान वेब पेज दिखाना

1. सामान्य रूप से अपना स्लाइडशो शुरू करें (**F5** या **Slide Show** टैब)
2. रिबन पर **Show WebPage.PPT** क्लिक करें — आपकी साइट के साथ एक ब्राउज़र विंडो खुलेगी
3. स्लाइड्स पर वापस लौटने के लिए विंडो बंद करें (या ऑटो-क्लोज़ टाइमर की प्रतीक्षा करें)

### भाषा बदलना

Web.PPT **10 भाषाओं** का समर्थन करता है: अंग्रेज़ी, चीनी (中文), स्पेनिश, जर्मन, फ़्रेंच, इतालवी, अरबी, पुर्तगाली, हिंदी, और रूसी। Task Pane हेडर में भाषा ड्रॉपडाउन का उपयोग करें — इंटरफ़ेस तुरंत अपडेट हो जाता है, रीस्टार्ट की आवश्यकता नहीं।

## अपनी खुद की साइट्स के लिए

Web.PPT वेबसाइटों को iframe के अंदर लोड करता है। अधिकांश व्यक्तिगत और प्रोजेक्ट साइट्स बिना किसी बदलाव के काम करती हैं। यदि आपकी साइट एम्बेडिंग ब्लॉक करती है, तो आपको समाधान के साथ एक अनुकूल संदेश दिखाई देगा।

अपने सर्वर कॉन्फ़िग में **एक लाइन** जोड़ें:

**Nginx**
```nginx
add_header Content-Security-Policy "frame-ancestors *";
```

**Apache** (`.htaccess` या config)
```apache
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
```

**Node.js / Express**
```javascript
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
```

बदलाव के बाद अपना सर्वर रीस्टार्ट करें। Web.PPT में एक बिल्ट-इन सेटअप गाइड भी है — जब कोई साइट ब्लॉक हो तो Task Pane में **"Site not loading?"** क्लिक करें।

**सर्वर एक्सेस नहीं है?** व्यूअर में **"Open directly"** क्लिक करें — यह iframe प्रतिबंधों के बिना पेज को पूर्ण ब्राउज़र विंडो में खोलता है।

## समर्थित प्लेटफ़ॉर्म

| प्लेटफ़ॉर्म | स्थिति | नोट्स |
|----------|--------|-------|
| **Windows 10/11** (Desktop) | समर्थित | शेयर्ड फ़ोल्डर Sideloading या AppSource के माध्यम से पूर्ण कार्यक्षमता |
| **Mac** (Desktop) | समर्थित | `~/Library/.../wef/` फ़ोल्डर या AppSource के माध्यम से Sideload |
| **iPad** | समर्थित | केवल AppSource के माध्यम से; dialog का व्यवहार भिन्न हो सकता है |
| **PowerPoint Online** | समर्थित | Upload के माध्यम से Sideload; स्लाइड बदलने पर ऑटो-ओपन उपलब्ध नहीं |

---

## ज्ञात सीमाएं

- **तृतीय-पक्ष साइट्स जो iframes ब्लॉक करती हैं** (उदा., Google, Facebook) को एम्बेड नहीं किया जा सकता। "Open directly" फ़ॉलबैक का उपयोग करें।
- **PowerPoint Online स्लाइडशो:** `ActiveViewChanged` ईवेंट ट्रिगर नहीं होता, इसलिए स्लाइड बदलने पर ऑटो-ओपन उपलब्ध नहीं है। रिबन बटन को मैन्युअली उपयोग करें।
- **Office 2019 और पुराने:** समर्थित नहीं — आवश्यक PowerPoint JavaScript API उपलब्ध नहीं है (देखें [सिस्टम आवश्यकताएं](#सिस्टम-आवश्यकताएं))।
- व्यूअर एक **अलग ब्राउज़र विंडो** में खुलता है, स्लाइड क्षेत्र के अंदर नहीं। यह Office Web Add-ins की प्लेटफ़ॉर्म सीमा है (विकल्प — Content Add-in — [स्लाइडशो मोड में टूटा हुआ है](https://github.com/OfficeDev/office-js/issues/4410))।

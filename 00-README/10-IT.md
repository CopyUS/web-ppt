# Web.PPT — Pagine Web Live in PowerPoint

> Mostra qualsiasi sito web durante la tua presentazione PowerPoint — completamente controllabile con un clicker, senza mai uscire dalla presentazione.

---

## Perché Web.PPT?

Mostrare una pagina web live durante una presentazione dovrebbe essere semplice. Purtroppo non lo è:

| Strumento | Problema |
|-----------|----------|
| **LiveWeb VBA** | Utilizzava il controllo ActiveX di Internet Explorer — dismesso da Microsoft. Inoltre catturava il focus di tastiera/mouse, rendendo impossibile controllare la presentazione con un clicker o telecomando. |
| **Microsoft Web Viewer** | Non funziona in modalità presentazione ([bug noti](https://github.com/OfficeDev/office-js/issues/4410), non corretti da anni) |
| **ClassPoint embedded browser** | Rimosso a ottobre 2025 |

**Web.PPT** è un componente aggiuntivo di Office gratuito e open-source che colma questa lacuna. Funziona ovunque PowerPoint sia disponibile — Windows, Mac, iPad e Web.

### Come funziona

1. Inserisci un URL nel pannello delle impostazioni (Task Pane)
2. Durante la presentazione, fai clic su **Show WebPage.PPT** sulla barra multifunzione
3. Si apre una finestra del browser sopra la presentazione con il sito web live
4. Il timer di chiusura automatica chiude la finestra del visualizzatore automaticamente dopo il tempo impostato, restituendo il focus a PowerPoint — così puoi continuare a navigare tra le diapositive con un clicker o telecomando

Nessuna macro VBA. Nessun ActiveX. Nessun problema di installazione.

## Requisiti di sistema

| Requisito | Dettagli |
|-----------|----------|
| **Versione di PowerPoint** | **Microsoft 365** (abbonamento) o **Office 2021** (acquisto una tantum) |
| **Sistema operativo** | Windows 10/11, macOS, iPadOS o qualsiasi sistema operativo con un browser (PowerPoint Online) |
| **Connessione Internet** | Necessaria per caricare le pagine web |

### Perché Office 2019 e versioni precedenti non sono supportati

Web.PPT utilizza la PowerPoint JavaScript API (`PowerPoint.run`, `getSelectedSlides`) per rilevare la diapositiva corrente e salvare le impostazioni per ogni diapositiva. Questa API (requirement set **PowerPointApi 1.1+**) è disponibile solo in:

- Microsoft 365 (tutti i canali di aggiornamento)
- Office 2021 (licenza perpetua)

Office 2019, 2016 e versioni precedenti non includono queste API, pertanto il componente aggiuntivo **non può determinare quale diapositiva è attiva** e non funzionerà correttamente.

## Installazione

### Opzione 1: AppSource (Microsoft 365)

> Questa opzione sarà eventualmente disponibile dopo la pubblicazione del componente aggiuntivo su AppSource.

Funziona su tutte le piattaforme (Windows, Mac, iPad, Web):

1. Apri PowerPoint → scheda **Inserisci** → **Ottieni componenti aggiuntivi**
2. Cerca **"Web.PPT"**
3. Fai clic su **Aggiungi** — il componente aggiuntivo appare immediatamente sulla barra multifunzione

---

### Opzione 2: Sideloading — PowerPoint 2021 / 365 Desktop (Windows)

Usa questo metodo per installare il componente aggiuntivo da un file locale o una cartella condivisa. Lo scenario base quando AppSource non è disponibile.

#### Passaggio 1: Abilitare la scheda Sviluppatore sulla barra multifunzione

La scheda Sviluppatore è necessaria per il Sideloading dei componenti aggiuntivi. Se non la vedi sulla barra multifunzione:

1. Apri PowerPoint → **File** → **Opzioni** → **Personalizza barra multifunzione**
2. Nella colonna di destra ("Schede principali"), seleziona la casella **Sviluppatore**
3. Fai clic su **OK**

La scheda **Sviluppatore** appare ora sulla barra multifunzione.

#### Passaggio 2: Scaricare il manifesto

Scarica `manifest-gh.xml` dalla [pagina dei rilasci](https://github.com/CopyUS) o dalla cartella `dist/` di questo repository.

#### Passaggio 3: Creare una cartella condivisa

1. Crea una cartella sul tuo computer, ad esempio: `C:\OfficeAddins\`
2. Inserisci `manifest-gh.xml` in questa cartella
3. Fai clic con il pulsante destro sulla cartella → **Proprietà** → scheda **Condivisione** → **Condividi...**
4. Condividi la cartella con te stesso (o mantieni le impostazioni di condivisione predefinite)
5. Copia il **percorso di rete** (ad esempio, `\\YOURPC\OfficeAddins`)

#### Passaggio 4: Aggiungere la cartella come Catalogo attendibile

1. Apri PowerPoint → **File** → **Opzioni** → **Centro protezione** → **Impostazioni Centro protezione...**
2. Seleziona **Cataloghi di componenti aggiuntivi attendibili** nel menu a sinistra
3. Nel campo **URL catalogo**, incolla il percorso di rete dal Passaggio 3 (ad esempio, `\\YOURPC\OfficeAddins`)
4. Fai clic su **Aggiungi catalogo**
5. Seleziona la casella **Mostra nel menu** per il catalogo aggiunto
6. Fai clic su **OK** → **OK** per chiudere tutte le finestre di dialogo
7. **Riavvia PowerPoint** (chiudi completamente tutte le finestre di PowerPoint)

#### Passaggio 5: Caricare il componente aggiuntivo

1. Riavvia PowerPoint (chiudi completamente e riapri)
2. Vai alla scheda **Sviluppo** → clicca su **Componenti aggiuntivi** (Add-ins)
3. Seleziona la scheda **Cartella condivisa** (Shared Folder) in alto
4. Vedrai **Web.PPT** — clicca su **Aggiungi** (Add)
5. Attendi qualche secondo — i pulsanti **Add WebPage.PPT** e **Show WebPage.PPT** appariranno nella scheda **Inserisci**

> **Importante:** I pulsanti **non appaiono automaticamente** sulla barra multifunzione dopo l'aggiunta della cartella attendibile. È necessario caricare manualmente il componente aggiuntivo tramite **Sviluppo → Componenti aggiuntivi → Cartella condivisa**. Questa è una caratteristica dell'installazione sideloading — è necessario aggiungere il componente aggiuntivo ogni volta che si riavvia PowerPoint. Dopo la pubblicazione su AppSource, il componente aggiuntivo verrà caricato automaticamente.

#### Se qualcosa è andato storto

| Problema | Soluzione |
|----------|-----------|
| Il componente aggiuntivo non appare in "Cartella condivisa" | Verifica il percorso di rete nelle impostazioni dei cataloghi attendibili. Assicurati che **"Mostra nel menu"** sia selezionato. Chiudi e riapri PowerPoint completamente. |
| La scheda "Cartella condivisa" è vuota | Chiudi **tutte** le finestre di PowerPoint, attendi 10 secondi e riapri. A volte sono necessari 2-3 riavvii. |
| "Add-in Error" quando si fa clic su un pulsante | Svuota la cache di Office (vedi la sezione pulizia cache qui sotto), riavvia PowerPoint e aggiungi di nuovo il componente aggiuntivo. |
| I pulsanti sono apparsi ma non rispondono | Attendi 5-10 secondi dopo il caricamento — Office scarica i file dal server. Verifica la connessione internet. |

### Opzione 3: Sideloading — PowerPoint Online

1. Vai su [office.com](https://www.office.com/) → apri PowerPoint → apri una presentazione qualsiasi
2. Scheda **Inserisci** → **Componenti aggiuntivi** → **Carica componente aggiuntivo personale**
3. Fai clic su **Sfoglia...** e seleziona il file `manifest.json` (formato JSON, non XML)
4. Fai clic su **Carica** — il componente aggiuntivo viene caricato immediatamente

> **Nota:** I componenti aggiuntivi caricati tramite Sideloading in PowerPoint Online persistono solo per la sessione corrente. Sarà necessario ricaricarli dopo aver aggiornato la pagina.

---

### Opzione 4: Sideloading — Mac (Desktop)

1. Scarica `manifest-gh.xml`
2. Apri Finder e naviga fino a: `~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/`
   > Se la cartella `wef` non esiste, creala manualmente.
3. Copia `manifest-gh.xml` nella cartella `wef`
4. Apri (o riavvia) PowerPoint
5. Vai su **Inserisci** → **Componenti aggiuntivi personali** → **La mia organizzazione** — dovresti vedere **Web.PPT**

> **Suggerimento:** Per aprire la cartella Library, premi **Cmd + Shift + G** in Finder e incolla il percorso.

## Svuotare la cache di Office (dopo gli aggiornamenti)

Quando aggiorni i file del componente aggiuntivo (nuovo `manifest-gh.xml` o pagine ospitate aggiornate), Office potrebbe ancora utilizzare la vecchia versione memorizzata nella cache. Svuota la cache per forzare un ricaricamento.

### Windows

1. Chiudi **tutte** le applicazioni Office (PowerPoint, Word, Excel, ecc.)
2. Apri Esplora file e naviga fino a: `%LOCALAPPDATA%\Microsoft\Office\16.0\Wef\`
   > **Suggerimento:** Premi **Win + R**, incolla il percorso e premi Invio.
3. Elimina **tutti i file e le cartelle** all'interno della cartella `Wef`
4. Riavvia PowerPoint e riaggiungi il componente aggiuntivo se necessario

### Mac

1. Chiudi tutte le applicazioni Office
2. Apri Finder → **Vai** → **Vai alla cartella...** (Cmd + Shift + G)
3. Naviga fino a: `~/Library/Containers/com.microsoft.Powerpoint/Data/Library/Caches/`
4. Elimina il contenuto della cartella `Caches`
5. Rimuovi e ricopia anche il manifesto in: `~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/`
6. Riavvia PowerPoint

### PowerPoint Online

1. Apri gli Strumenti per sviluppatori del browser (**F12**)
2. Vai alla scheda **Application** → **Storage** → **Clear site data**
3. Aggiorna la pagina e ricarica il manifesto

## Utilizzo

### Impostare un URL per una diapositiva

Ogni diapositiva può avere il proprio URL. Seleziona una diapositiva, apri il Task Pane (**Inserisci** → **Add WebPage.PPT**) e inserisci l'URL.

| Impostazione | Descrizione | Predefinito |
|--------------|-------------|-------------|
| **URL** | L'indirizzo web da visualizzare | *(nessuno)* |
| **Zoom** | Scala della pagina (50 %–300 %) | 100 % |
| **Larghezza finestra** | Larghezza della finestra del visualizzatore (30 %–100 % dello schermo) | 80 % |
| **Altezza finestra** | Altezza della finestra del visualizzatore (30 %–100 % dello schermo) | 80 % |
| **Apertura automatica** | Apri automaticamente il visualizzatore quando questa diapositiva viene mostrata | Disattivato |
| **Chiusura automatica** | Chiudi automaticamente il visualizzatore dopo un tempo impostato (0–3600 s) | Disattivato |

Fai clic su **Imposta come predefiniti** per salvare le dimensioni/zoom preferiti per tutte le nuove diapositive.

### Mostrare una pagina web durante una presentazione

1. Avvia la presentazione normalmente (**F5** o scheda **Presentazione**)
2. Fai clic su **Show WebPage.PPT** sulla barra multifunzione — si apre una finestra del browser con il tuo sito
3. Chiudi la finestra (o attendi il timer di chiusura automatica) per tornare alle diapositive

### Cambiare la lingua

Web.PPT supporta **10 lingue**: inglese, cinese, spagnolo, tedesco, francese, italiano, arabo, portoghese, hindi e russo. Usa il menu a tendina della lingua nell'intestazione del Task Pane — l'interfaccia si aggiorna istantaneamente, senza bisogno di riavvio.

## Per i tuoi siti

Web.PPT carica i siti web all'interno di un iframe. La maggior parte dei siti personali e di progetto funziona immediatamente. Se il tuo sito blocca l'incorporamento, vedrai un messaggio informativo con la soluzione.

Aggiungi **una sola riga** alla configurazione del tuo server:

**Nginx**
```nginx
add_header Content-Security-Policy "frame-ancestors *";
```

**Apache** (`.htaccess` o configurazione)
```apache
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
```

**Node.js / Express**
```javascript
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
```

Riavvia il server dopo la modifica. Web.PPT include anche una guida alla configurazione integrata — fai clic su **"Il sito non si carica?"** nel Task Pane quando un sito è bloccato.

**Non hai accesso al server?** Fai clic su **"Apri direttamente"** nel visualizzatore — apre la pagina in una finestra del browser completa senza restrizioni iframe.

## Limitazioni note

- **I siti di terze parti che bloccano gli iframe** (ad esempio Google, Facebook) non possono essere incorporati. Usa il fallback "Apri direttamente".
- **Presentazione in PowerPoint Online:** L'evento `ActiveViewChanged` non viene attivato, quindi l'apertura automatica al cambio diapositiva non è disponibile. Usa il pulsante sulla barra multifunzione manualmente.
- **Office 2019 e versioni precedenti:** Non supportati — la PowerPoint JavaScript API necessaria non è disponibile (vedi [Requisiti di sistema](#requisiti-di-sistema)).
- Il visualizzatore si apre in una **finestra del browser separata**, non all'interno dell'area della diapositiva. Questa è una limitazione della piattaforma dei componenti aggiuntivi web di Office (l'alternativa — Content Add-in — è [non funzionante in modalità presentazione](https://github.com/OfficeDev/office-js/issues/4410)).

# Web.PPT — 在 PowerPoint 中展示实时网页

> 在 PowerPoint 演示期间展示任意网站 — 使用翻页器即可完全控制，无需离开幻灯片放映。

---

## 为什么选择 Web.PPT？

在演示过程中展示实时网页本应很简单。遗憾的是，事实并非如此：

| 工具 | 问题 |
|------|------|
| **LiveWeb VBA** | 使用了 Internet Explorer ActiveX 控件 — 已被 Microsoft 停止支持。此外，LiveWeb 会捕获键盘/鼠标焦点，导致无法使用翻页器或遥控器控制演示。 |
| **Microsoft Web Viewer** | 在幻灯片放映模式下无法正常工作（[已知缺陷](https://github.com/OfficeDev/office-js/issues/4410)，多年未修复） |
| **ClassPoint embedded browser** | 已于 2025 年 10 月移除 |

**Web.PPT** 是一个免费的开源 Office 加载项，填补了这一空白。它可以在 PowerPoint 运行的所有平台上工作 — Windows、Mac、iPad 和 Web。

### 工作原理

1. 在设置面板 (Task Pane) 中输入 URL
2. 在幻灯片放映期间，点击功能区上的 **Show WebPage.PPT**
3. 演示文稿上方会打开一个浏览器窗口，显示实时网站
4. 自动关闭计时器会在设定时间后自动关闭查看窗口，将焦点返回给 PowerPoint — 您可以继续使用翻页器或遥控器切换幻灯片

无需 VBA 宏。无需 ActiveX。无需复杂安装。

## 系统要求

| 要求 | 详情 |
|------|------|
| **PowerPoint 版本** | **Microsoft 365**（订阅版）或 **Office 2021**（永久许可证） |
| **操作系统** | Windows 10/11、macOS、iPadOS，或任何带浏览器的操作系统 (PowerPoint Online) |
| **网络连接** | 加载网页时需要 |

### 为什么不支持 Office 2019 及更早版本

Web.PPT 使用 PowerPoint JavaScript API（`PowerPoint.run`、`getSelectedSlides`）来检测当前幻灯片并保存每张幻灯片的设置。该 API（requirement set **PowerPointApi 1.1+**）仅在以下版本中可用：

- Microsoft 365（所有更新通道）
- Office 2021（永久许可证）

Office 2019、2016 及更早版本不包含这些 API，因此加载项**无法确定当前活动的幻灯片**，将无法正常工作。

## 安装

### 方式 1：AppSource (Microsoft 365)

> 此方式可能在加载项发布到 AppSource 后可用。

适用于所有平台（Windows、Mac、iPad、Web）：

1. 打开 PowerPoint → **插入**选项卡 → **获取加载项**
2. 搜索 **"Web.PPT"**
3. 点击**添加** — 加载项会立即出现在功能区上

---

### 方式 2：Sideloading — PowerPoint 2021 / 365 Desktop (Windows)

使用此方法从本地文件或共享文件夹安装加载项。这是 AppSource 不可用时的基本方案。

#### 步骤 1：在功能区启用"开发工具"选项卡

"开发工具"选项卡是 Sideloading 加载项所必需的。如果功能区上没有该选项卡：

1. 打开 PowerPoint → **文件** → **选项** → **自定义功能区**
2. 在右侧列（"主选项卡"）中，勾选**开发工具**复选框
3. 点击**确定**

**开发工具**选项卡现在会出现在功能区上。

#### 步骤 2：下载清单文件

从[发布页面](https://github.com/CopyUS)或本仓库的 `dist/` 文件夹下载 `manifest-gh.xml`。

#### 步骤 3：创建共享文件夹

1. 在计算机上创建一个文件夹，例如：
   ```
   C:\OfficeAddins\
   ```
2. 将 `manifest-gh.xml` 放入此文件夹
3. 右键点击文件夹 → **属性** → **共享**选项卡 → **共享...**
4. 与自己共享文件夹（或保留默认共享设置）
5. 复制**网络路径**（例如 `\\YOURPC\OfficeAddins`）

#### 步骤 4：将文件夹添加为受信任的目录

1. 打开 PowerPoint → **文件** → **选项** → **信任中心** → **信任中心设置...**
2. 在左侧菜单中选择**受信任的加载项目录**
3. 在**目录 URL** 字段中，粘贴步骤 3 中的网络路径（例如 `\\YOURPC\OfficeAddins`）
4. 点击**添加目录**
5. 为添加的目录勾选**在菜单中显示**复选框
6. 点击**确定** → **确定**以关闭所有对话框
7. **重新启动 PowerPoint**（完全关闭所有 PowerPoint 窗口）

#### 步骤 5：加载加载项

1. 重新启动 PowerPoint（完全关闭后重新打开）
2. 转到 **开发工具** 选项卡 → 点击 **加载项** (Add-ins)
3. 选择顶部的 **共享文件夹** (Shared Folder) 选项卡
4. 您将看到 **Web.PPT** — 点击 **添加** (Add)
5. 等待几秒钟 — **Add WebPage.PPT** 和 **Show WebPage.PPT** 按钮将出现在 **插入** 选项卡上

> **重要：** 添加受信任文件夹后，按钮**不会自动出现**在功能区上。您需要通过 **开发工具 → 加载项 → 共享文件夹** 手动加载加载项。这是旁加载安装的特性 — 每次重启 PowerPoint 都需要重新添加。发布到 AppSource 后，加载项将自动加载。

#### 如果出现问题

| 问题 | 解决方案 |
|------|----------|
| 加载项未显示在"共享文件夹"中 | 检查受信任目录设置中的网络路径。确保勾选了**"在菜单中显示"**。完全关闭并重新打开 PowerPoint。 |
| "共享文件夹"选项卡为空 | 关闭**所有** PowerPoint 窗口，等待10秒，重新打开。有时需要2-3次重启。 |
| 点击按钮时出现"Add-in Error" | 清除 Office 缓存（参见下方缓存清除部分），重启 PowerPoint 并重新添加加载项。 |
| 按钮出现但没有反应 | 加载后等待5-10秒 — Office 正在从服务器下载文件。检查网络连接。 |

### 方式 3：Sideloading — PowerPoint Online

1. 前往 [office.com](https://www.office.com/) → 打开 PowerPoint → 打开任意演示文稿
2. **插入**选项卡 → **加载项** → **上传我的加载项**
3. 点击**浏览...**，选择 `manifest.json` 文件（JSON 格式，非 XML）
4. 点击**上传** — 加载项会立即加载

> **注意：** 在 PowerPoint Online 中通过 Sideloading 加载的加载项仅在当前会话中有效。刷新页面后需要重新上传。

---

### 方式 4：Sideloading — Mac (Desktop)

1. 下载 `manifest-gh.xml`
2. 打开 Finder 并导航到：
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
   > 如果 `wef` 文件夹不存在，请手动创建。
3. 将 `manifest-gh.xml` 复制到 `wef` 文件夹中
4. 打开（或重新启动）PowerPoint
5. 前往**插入** → **我的加载项** → **我的组织** — 您应该能看到 **Web.PPT**

> **提示：** 要打开 Library 文件夹，请在 Finder 中按 **Cmd + Shift + G** 并粘贴路径。

## 清除 Office 缓存（更新后）

当您更新加载项文件（新的 `manifest-gh.xml` 或更新的托管页面）时，Office 可能仍然使用缓存中的旧版本。清除缓存以强制重新加载。

### Windows

1. 关闭**所有** Office 应用程序（PowerPoint、Word、Excel 等）
2. 打开文件资源管理器并导航到：
   ```
   %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\
   ```
   > **提示：** 按 **Win + R**，粘贴路径，然后按 Enter。
3. 删除 `Wef` 文件夹内的**所有文件和文件夹**
4. 重新启动 PowerPoint，如有需要重新添加加载项

### Mac

1. 关闭所有 Office 应用程序
2. 打开 Finder → **前往** → **前往文件夹...** (Cmd + Shift + G)
3. 导航到：
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Library/Caches/
   ```
4. 删除 `Caches` 文件夹中的内容
5. 同时删除并重新复制清单文件到：
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
6. 重新启动 PowerPoint

### PowerPoint Online

1. 打开浏览器开发者工具 (**F12**)
2. 前往 **Application** 选项卡 → **Storage** → **Clear site data**
3. 刷新页面并重新上传清单文件

## 使用方法

### 为幻灯片设置 URL

每张幻灯片可以有自己的 URL。选择一张幻灯片，打开 Task Pane（**插入** → **Add WebPage.PPT**），然后输入 URL。

可用设置：

| 设置 | 说明 | 默认值 |
|------|------|--------|
| **URL** | 要显示的网页地址 | *（无）* |
| **缩放** | 页面缩放比例 (50 %–300 %) | 100 % |
| **窗口宽度** | 查看窗口宽度 (30 %–100 % 屏幕) | 80 % |
| **窗口高度** | 查看窗口高度 (30 %–100 % 屏幕) | 80 % |
| **自动打开** | 切换到该幻灯片时自动打开查看窗口 | 关闭 |
| **自动关闭** | 在设定时间后自动关闭查看窗口 (0–3600 秒) | 关闭 |

点击**设为默认值**可将当前的尺寸/缩放设置保存为所有新幻灯片的默认值。

### 在幻灯片放映期间展示网页

1. 按照正常方式开始幻灯片放映（**F5** 或**幻灯片放映**选项卡）
2. 点击功能区上的 **Show WebPage.PPT** — 浏览器窗口会打开并显示您的网站
3. 关闭窗口（或等待自动关闭计时器）以返回幻灯片

### 更改语言

Web.PPT 支持 **10 种语言**：英语、中文 (中文)、西班牙语、德语、法语、意大利语、阿拉伯语、葡萄牙语、印地语和俄语。使用 Task Pane 标题中的语言下拉列表 — 界面会即时更新，无需重新启动。

## 针对您自己的网站

Web.PPT 在 iframe 中加载网站。大多数个人和项目网站无需额外配置即可正常工作。如果您的网站阻止嵌入，您将看到一条友好的提示信息及修复方法。

在服务器配置中添加**一行**代码：

**Nginx**
```nginx
add_header Content-Security-Policy "frame-ancestors *";
```

**Apache** (`.htaccess` 或配置文件)
```apache
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
```

**Node.js / Express**
```javascript
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
```

修改后重新启动服务器。Web.PPT 还内置了设置指南 — 当网站被阻止时，在 Task Pane 中点击**"网站无法加载？"**。

**没有服务器访问权限？** 在查看窗口中点击**"直接打开"** — 页面将在完整的浏览器窗口中打开，不受 iframe 限制。

## 已知限制

- **阻止 iframe 的第三方网站**（如 Google、Facebook）无法嵌入。请使用"直接打开"功能。
- **PowerPoint Online 幻灯片放映：** `ActiveViewChanged` 事件不会触发，因此幻灯片切换时自动打开不可用。请手动使用功能区按钮。
- **Office 2019 及更早版本：** 不支持 — 所需的 PowerPoint JavaScript API 不可用（参见[系统要求](#系统要求)）。
- 查看窗口在**单独的浏览器窗口**中打开，而非嵌入幻灯片区域内。这是 Office Web Add-ins 的平台限制（替代方案 — Content Add-in — [在幻灯片放映模式下无法正常工作](https://github.com/OfficeDev/office-js/issues/4410)）。

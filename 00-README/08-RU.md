# Web.PPT — Живые веб-страницы в PowerPoint

> Показывайте любой сайт прямо во время презентации — управляйте показом с кликера, не покидая слайдшоу.

---

## Зачем Web.PPT?

Показать живую веб-страницу во время презентации должно быть просто. К сожалению, это не так:

| Инструмент | Проблема |
|------------|---------|
| **LiveWeb VBA** | Использовал ActiveX-элемент Internet Explorer — прекращена поддержка Microsoft. Кроме того,LiveWeb перехватывал фокус клавиатуры/мыши, делая невозможным управление презентацией с кликера или пульта. |
| **Microsoft Web Viewer** | Не работает в режиме показа слайдов ([известные баги](https://github.com/OfficeDev/office-js/issues/4410), не работает уже годы) |
| **ClassPoint embedded browser** | Удалён в октябре 2025 |

**Web.PPT** — бесплатное open-source расширение для Office, которое решает эту проблему. Работает везде, где есть PowerPoint — Windows, Mac, iPad и Web.

### Как это работает

1. Вы вводите URL в панели настроек (Task Pane)
2. Во время показа слайдов нажимаете **Show WebPage.PPT** на ленте
3. Поверх презентации открывается окно браузера с живым сайтом
4. Таймер автозакрытия закрывает окно просмотра автоматически через заданное время, возвращая фокус PowerPoint — и вы продолжаете переключать слайды кликером или пультом

Без VBA-макросов. Без ActiveX. Без сложной установки.

## Системные требования

| Требование | Детали |
|------------|--------|
| **Версия PowerPoint** | **Microsoft 365** (подписка) или **Office 2021** (бессрочная лицензия) |
| **Операционная система** | Windows 10/11, macOS, iPadOS, или любая ОС с браузером (PowerPoint Online) |
| **Интернет-соединение** | Необходимо для загрузки веб-страниц |

### Почему Office 2019 и более ранние версии не поддерживаются

Web.PPT использует PowerPoint JavaScript API (`PowerPoint.run`, `getSelectedSlides`) для определения текущего слайда и сохранения настроек по слайдам. Этот API (requirement set **PowerPointApi 1.1+**) доступен только в:

- Microsoft 365 (все каналы обновлений)
- Office 2021 (бессрочная лицензия)

Office 2019, 2016 и более ранние версии не включают эти API, поэтому аддон **не может определить активный слайд** и не будет работать корректно.

## Установка

### Способ 1: AppSource (Microsoft 365)

> Этот способ возможно когда-то будет доступен после публикации аддона в AppSource.

Работает на всех платформах (Windows, Mac, iPad, Web):

1. Откройте PowerPoint → вкладка **Вставка** → **Получить надстройки**
2. Найдите **"Web.PPT"**
3. Нажмите **Добавить** — аддон сразу появится на ленте

---

### Способ 2: Sideloading — PowerPoint 2021 / 365 Desktop (Windows)

Используйте этот метод для установки из локального файла или общей папки. Базовый сценарий когда AppSource недоступен.

#### Шаг 1: Включите вкладку «Разработчик» на ленте

Вкладка «Разработчик» необходима для загрузки аддонов. Если её нет на ленте:

1. Откройте PowerPoint → **Файл** → **Параметры** → **Настроить ленту**
2. В правой колонке («Основные вкладки») поставьте галочку напротив **Разработчик**
3. Нажмите **OK**

Вкладка **Разработчик** теперь отображается на ленте.

#### Шаг 2: Скачайте манифест

Скачайте `manifest-gh.xml` со [страницы релизов](https://github.com/CopyUS) или из папки `dist/` этого репозитория.

#### Шаг 3: Создайте общую папку

1. Создайте папку на компьютере, например:
   ```
   C:\OfficeAddins\
   ```
2. Поместите `manifest-gh.xml` в эту папку
3. Щёлкните правой кнопкой по папке → **Свойства** → вкладка **Доступ** → **Поделиться...**
4. Предоставьте доступ самому себе (или оставьте настройки по умолчанию)
5. Скопируйте **сетевой путь** (например, `\\YOURPC\OfficeAddins`)

#### Шаг 4: Добавьте папку как доверенный каталог

1. Откройте PowerPoint → **Файл** → **Параметры** → **Центр управления безопасностью** → **Параметры центра управления безопасностью...**
2. Выберите **Доверенные каталоги надстроек** в левом меню
3. В поле **URL-адрес каталога** вставьте сетевой путь из Шага 3 (например, `\\YOURPC\OfficeAddins`)
4. Нажмите **Добавить каталог**
5. Поставьте галочку **Показывать в меню** для добавленного каталога
6. Нажмите **OK** → **OK** для закрытия всех диалогов
7. **Перезапустите PowerPoint** (закройте все окна PowerPoint полностью)

#### Шаг 5: Загрузите аддон

1. Откройте PowerPoint → вкладка **Вставка** → **Получить надстройки** (или **Мои надстройки**)
2. Выберите вкладку **Общая папка** вверху
3. Вы увидите **Web.PPT** — нажмите **Добавить**
4. Кнопки аддона появятся на вкладке **Вставка**

> **Совет:** Если аддон не отображается в «Общей папке», проверьте сетевой путь и перезапустите PowerPoint.

---

### Способ 3: Sideloading — PowerPoint Online

1. Перейдите на [office.com](https://www.office.com/) → откройте PowerPoint → откройте любую презентацию
2. Вкладка **Вставка** → **Надстройки** → **Отправить мою надстройку**
3. Нажмите **Обзор...** и выберите файл `manifest.json` (формат JSON, не XML)
4. Нажмите **Отправить** — аддин загрузится сразу

> **Примечание:** Загруженные аддоны в PowerPoint Online сохраняются только на время текущей сессии. После обновления страницы потребуется повторная загрузка.

---

### Способ 4: Sideloading — Mac (Desktop)

1. Скачайте `manifest-gh.xml`
2. Откройте Finder и перейдите по пути:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
   > Если папки `wef` нет — создайте её вручную.
3. Скопируйте `manifest-gh.xml` в папку `wef`
4. Откройте (или перезапустите) PowerPoint
5. Перейдите в **Вставка** → **Мои надстройки** → **Моя организация** — вы увидите **Web.PPT**

> **Совет:** Чтобы открыть папку Library, нажмите **Cmd + Shift + G** в Finder и вставьте путь.

## Очистка кэша Office (после обновлений)

При обновлении файлов аддина (новый `manifest-gh.xml` или обновлённые страницы на хостинге) Office может продолжать использовать старую версию из кэша. Очистите кэш, чтобы принудительно обновить.

### Windows

1. Закройте **все** приложения Office (PowerPoint, Word, Excel и т.д.)
2. Откройте Проводник и перейдите по пути:
   ```
   %LOCALAPPDATA%\Microsoft\Office\16.0\Wef\
   ```
   > **Совет:** Нажмите **Win + R**, вставьте путь и нажмите Enter.
3. Удалите **все файлы и папки** внутри папки `Wef`
4. Перезапустите PowerPoint и при необходимости добавьте аддин заново

### Mac

1. Закройте все приложения Office
2. Откройте Finder → **Переход** → **Переход к папке...** (Cmd + Shift + G)
3. Перейдите по пути:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Library/Caches/
   ```
4. Удалите содержимое папки `Caches`
5. Также удалите и заново скопируйте манифест в:
   ```
   ~/Library/Containers/com.microsoft.Powerpoint/Data/Documents/wef/
   ```
6. Перезапустите PowerPoint

### PowerPoint Online

1. Откройте DevTools в браузере (**F12**)
2. Перейдите на вкладку **Application** → **Storage** → **Clear site data**
3. Обновите страницу и повторно загрузите манифест

## Использование

### Настройка URL для слайда

Каждый слайд может иметь свой URL. Выберите слайд, откройте Task Pane (**Вставка** → **Add WebPage.PPT**) и введите URL.

Доступные настройки:

| Настройка | Описание | По умолчанию |
|-----------|----------|-------------|
| **URL** | Адрес веб-страницы | *(нет)* |
| **Масштаб** | Масштабирование страницы (50 %–300 %) | 100 % |
| **Ширина окна** | Ширина окна просмотра (30 %–100 % экрана) | 80 % |
| **Высота окна** | Высота окна просмотра (30 %–100 % экрана) | 80 % |
| **Автооткрытие** | Автоматически открывать просмотр при переходе на слайд | Выкл |
| **Автозакрытие** | Автоматически закрывать окно через заданное время (0–3600 с) | Выкл |

Нажмите **Сохранить как настройки по умолчанию** для применения текущих размеров/масштаба ко всем новым слайдам.

### Показ веб-страницы во время презентации

1. Запустите показ слайдов как обычно (**F5** или вкладка **Показ слайдов**)
2. Нажмите **Show WebPage.PPT** на ленте — откроется окно браузера с вашим сайтом
3. Закройте окно (или дождитесь таймера автозакрытия) для возврата к слайдам

### Смена языка

Web.PPT поддерживает **10 языков**: английский, китайский (中文), испанский, немецкий, французский, итальянский, арабский, португальский, хинди и русский. Используйте выпадающий список языков в заголовке Task Pane — интерфейс обновляется мгновенно, без перезагрузки.

## Для ваших собственных сайтов

Web.PPT загружает сайты внутри iframe. Большинство личных и проектных сайтов работают без дополнительных настроек. Если ваш сайт блокирует встраивание, вы увидите понятное сообщение с инструкцией.

Добавьте **одну строку** в конфигурацию сервера:

**Nginx**
```nginx
add_header Content-Security-Policy "frame-ancestors *";
```

**Apache** (`.htaccess` или конфиг)
```apache
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
```

**Node.js / Express**
```javascript
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
```

Перезапустите сервер после изменения. В Web.PPT также есть встроенная инструкция — нажмите **«Сайт не загружается?»** в Task Pane, когда сайт заблокирован.

**Нет доступа к серверу?** Нажмите **«Открыть напрямую»** в окне просмотра — страница откроется в полном окне браузера без ограничений iframe.

## Поддерживаемые платформы

| Платформа | Статус | Примечания |
|-----------|--------|------------|
| **Windows 10/11** (Desktop) | Поддерживается | Полная функциональность через общую папку или AppSource |
| **Mac** (Desktop) | Поддерживается | Sideload через `~/Library/.../wef/` или AppSource |
| **iPad** | Поддерживается | Только через AppSource; поведение диалога может отличаться |
| **PowerPoint Online** | Поддерживается | Sideload через Upload; автооткрытие при смене слайда недоступно |

## Известные ограничения

- **Сторонние сайты, блокирующие iframe** (например, Google, Facebook) не могут быть встроены. Используйте функцию «Открыть напрямую».
- **PowerPoint Online в режиме показа:** Событие `ActiveViewChanged` не срабатывает, поэтому автооткрытие при смене слайда недоступно. Используйте кнопку на ленте вручную.
- **Office 2019 и старше:** Не поддерживается — необходимый PowerPoint JavaScript API недоступен (см. [Системные требования](#системные-требования)).
- Окно просмотра открывается в **отдельном окне браузера**, а не внутри слайда. Это ограничение платформы Office Web Add-ins (альтернатива — Content Add-in — [не работает в режиме показа](https://github.com/OfficeDev/office-js/issues/4410)).

How to allow embedding

Option 1: Link to an image

If your site can export content as an image (.png, .jpg, .webp, .gif, .svg), paste the direct URL to the image file. No server changes needed.

Option 2: Allow iframe embedding

Add one of these snippets to the server that hosts the linked page:

add_header Content-Security-Policy "frame-ancestors *";
Header set Content-Security-Policy "frame-ancestors *"
Header unset X-Frame-Options
app.use((req, res, next) => {
  res.setHeader('Content-Security-Policy', 'frame-ancestors *');
  res.removeHeader('X-Frame-Options');
  next();
});
<meta http-equiv="Content-Security-Policy"
      content="frame-ancestors *">

Note: frame-ancestors in a meta tag may be ignored if the server already sets X-Frame-Options headers.

Restart your server and reload the slide after making changes.

FAQ

What is X-Frame-Options?

An HTTP header that controls whether your site can be shown inside an iframe.

I don't know which server I have

Check your project files: nginx.conf, .htaccess, app.js...

I don't have server access

Use the "Open directly" button in the viewer.
